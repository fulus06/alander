方案讨论：下一步核心任务
我建议接下来的具体实施步骤如下：

1. 深度集成 ECS 与 渲染管线
现状：目前 Renderer 内部维护了一套渲染对象，而 SceneManager (ECS) 维护了一套实体，两者还是脱节的。
任务：修改模型加载逻辑，使 glTF 的每个 Mesh 节点都对应一个 ECS Entity。在 Entity 中存储对应的渲染对象 ID（UUID）。
2. 场景大纲视图 (Scene Outliner)
任务：在左侧面板中展示真实的实体树状结构。
功能：点击大纲中的实体名，系统记录“当前选中项”。
3. 实时属性面板 (Property Inspector)
任务：在右侧面板显示选中实体的 Transform 组件。
功能：实现数值输入框，支持实时修改坐标、旋转和缩放，并让 3D 视图中的模型同步变化。
4. 高级交互：物体拾取 (Ray-casting Picking)
任务：实现鼠标左键点击 3D 空间，能够通过射线检测精准选中对应的模型。
功能：被选中的物体在 3D 视图中显示高亮（如边界框或线框）。
您想先从哪一部分开始？

选项 A：同步 ECS 实体并展示场景大纲（建立数据基础）。
选项 B：实现属性面板并支持修改坐标（建立交互闭环）。
选项 C：直接挑战射线检测拾取功能（最核心的 3D 交互）。
